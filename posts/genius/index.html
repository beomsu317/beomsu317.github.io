Beomsu Lee May 1 2021-05-01T00:00:00+09:00
7 min

Problem

i번 곡 다음에 j번 곡이 재생될 확률을 나타내는 확률 T가 존재하며, 방금 0번 곡을 듣기 시작했다면, K분 30초가 지난 후 좋아하는 곡이 재생될 확률을 구하면 된다. 곡들의 길이는 1분, 2분, 3분, 4분이다.

  1. 입력의 첫 줄에는 테스트 케이스의 수 C(1 <= C <= 50)
  2. 각 테스트 케이스의 첫 줄에는 MP3 플레이어에 들어있는 곡의 수 N(1 <= N <= 50), K(1 <= K <= 1,000,000), 좋아하는 곡의 수 M(1 <= M <= 10)
  3. 그 다음 N개의 정수로 각 곡의 길이가 주어짐
  4. 그 후 N 줄에는 한 곡이 재생된 후 다음 곡이 재생될 확률을 나타내는 행렬 T가 주어짐 (0 <= i,j < N)
  5. 마지막 줄에는 M개의 정수로 좋아하는 곡의 번호가 주어짐

Solving

이 문제는 마르코프 연쇄를 다루는 문제이다. 다음과 같이 부분 문제를 점화식으로 표현할 수 있다.

start(time,song) = 재생을 시작한지 time 분 후 song 번 노래가 재생을 시작할 확률

song번 노래가 재생되기 전 노래를 prev라 하면 time - length[prev]분에 prev 노래가 재생을 시작해야 한다. 이것을 다음과 같은 점화식으로 표현할 수 있다.

$ start(time,song) = \sum_{prev} (start(time - length[prev],prev) * T([prev][song])) $

이를 다음과 같이 구현할 수 있다.

vector<double> getProb1(){
    // c[time][song] : time 분 후에 song번 노래기 시작될 확률
    double c[5][50];
    memset(c,0,sizeof(c));
    c[0][0] = 1.0;
    for(int time = 1; time <= k; time++){
        for(int song = 0; song < n;song++){
            double &prob = c[time % 5][song];
            prob = 0;
            for(int last = 0 ; last < n ;last++){
                // +5 하는 이유는 -를 고려한 것이고 5인 이유는 0이 노래 길이에 0이 포함되지 않기 때문
                prob += c[(time - length[last] + 5) % 5][last] * T[last][song];
            }
        }
    }
    vector<double> ret(n);
    // song 번 노래가 재생될 확률 계산
    for(int song = 0; song < n; song++){
        // song 번 노래가 시작했을 시간을 모두 더한다.
        // +1 한 이유는 k분 30초인 것을 고려
        for(int start = k - length[song] + 1 ; start <= k ; start++){
            ret[song] += c[start % 5][song];
        }   
    }
    return ret;
}

하지만 위의 방식으로 풀 경우 시간복잡도가 $O(n^2k)$이며 k가 너무 크기 때문에 시간초과가 발생한다. 따라서 행렬 거듭제곱을 이용한 동적 계획법을 사용해야 한다. start(time,song)을 표현하는 점화식은 start() 값의 선형 결합 형태를 갖고 있기 때문에 해당 기법을 사용할 수 있다.

start(time - 3)부터 start(time)까지 포함하는 크기 4xN 열벡터 $ C_{time} $을 다음과 같이 정의할 수 있다.

\[C_{time} = { \begin{bmatrix} start(time-3,0) ... start(time-3,n-1) \cr start(time-2,0) ... start(time-2,n-1) \cr start(time-1,0) ... start(time-1,n-1) \cr start(time,0) ... start(time,n-1) \cr \end{bmatrix} }\]

열벡터 C를 이용해 $C(i)$에서 $C(i+1)$로 가기 위한 가중치를 저장하는 행렬 4*N x 4*N 정사각행렬을 만들 수 있다.

(4-1)*N 까지는 이전에 시작했던 노래들보다 1분 늦게 시작한 노래들을 그대로 가져온다.

그리고 (4-1)*N 다음부턴 다음 공식을 이용해 만들 수 있다.

C[time + 1][i] = C[time + 1 - length[j][j] * T[j][i]

$C_{time}$만 있으면 $C_{time+1}$을 만들 수 있다. $C_{time} = W * C_{time}$가 성립하는 행렬 W만 있으면 이 문제를 풀 수 있다. 행렬의 거듭제곱으로 $W^k$를 구한 뒤, $C(0)$과 곱한 결과를 계산하면 된다.

vector<double> getProb(void){
    SquareMatrix W(4 * n);

    for (int i = 0; i < 3 * n; i++)
        W[i][i + n] = 1.0;

    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            W[3 * n + i][(4 - length[j]) * n + j] = T[j][i];

    SquareMatrix Wk = W.pow(K); 
    vector<double> result(n);

    // song번 노래가 재생될 확률 계산
    for (int song = 0; song < n; song++)
        // song번 노래가 시작했을 시간을 모두 찾아 더함
        for (int start = 0; start < length[song]; start++)
            result[song] += Wk[(3 - start) * n + song][3 * n];

    return result;
}

노래의 최대 길이가 l이라고 할 때, 정방 행렬 W의 크기는 nl이므로 수행 시간은 $O((nl)^3lgk)$이다.
