Choose the Order of Match

Beomsu Lee May 3 2021-05-03T00:00:00+09:00
4 min

Problem

한국팀과 러시아팀이 프로그래밍 대회 결승전을 치룬다. 각 선수가 한 번씩 출전해 1:1 경기를 하여 더 많은 승리를 가져가는 팀이 우승한다. 각 팀은 대회 전날 주최측에 각 출전시킬 선수의 순서를 알려줘야 한다. 결승전 이틀 전, 한국팀이 러시아팀의 출전 순서를 알아냈다. 1:1 승부에선 항상 레이팅이 높은 선수가 승리하고, 같을 경우 한국팀 선수가 승리한다고 가정한다.

경기123456
러시아팀300027002800220025001900
한국팀280027502995180026002000

위와 같이 출전 순서를 정했을 경우, 한국팀은 2번, 3번, 5번, 6번 경기에서 승리해 4 경기를 이기게 된다. 4번 경기와 1번 경기에 나갈 선수를 바꾸면 1번 경기만을 제외하고 모든 경기에 승리할 수 있다. 상대팀 선수들의 순서를 알고 있을 때, 승수를 최대화하는 프로그램을 만들면 된다.

  1. 입력의 첫 줄에는 테스트 케이스의 수 C(C <= 50)
  2. 각 테스트 케이스의 첫 줄에는 각 팀 선수의 수 N(1 <= N <= 100)
  3. 그 다음 줄에는 N개의 정수로 러시아팀 각 선수의 레이팅이 주어지며, 그 다음 줄에는 N개의 정수로 한국팀 각 선수의 레이팅이 무순으로 주어짐
  4. 모든 레이팅 1 이상 4000 이하의 정수

Solving

n명의 선수가 있으니 n!개의 답이 존재한다. n!의 순열을 만드는 것은 완전 탐색을 통해 만들 순 있지만 n이 조금만 커져도 답의 개수가 많아지기 때문에 시간 안에 다 셀 수 없다.

메모제이션을 적용해 다음과 같이 동적 계획법을 구현할 수 있다. $O(n2^n)$ 시간의 걸린다.

order(taken) = 각 한국팀 선수를 이미 순서에 추가했는지 여부가 taken에 주어질 때, 남은 선수들을 적절히 배치해 얻을 수 있는 최대 승수

하지만 100명의 선수들로 구성되어 있다면 더 빠른 알고리즘으로 구현해야 한다. 탐욕 알고리즘을 사용하면 시간 안에 풀이가 가능하다.

상대방 선수를 이길 수 있는 한국 선수가 있는 경우 그 중 레이팅이 가장 낮은 선수를 상대방 선수와 경기시킨다. 만약 상대방 선수가 한국팀의 모든 선수보다 레이팅이 높다면 남은 선수 중 가장 레이팅이 낮은 선수와 경기시킨다. 이와 같은 방식으로 구현하면 최대 승수를 구할 수 있다.

int order(vector<int> &russian,vector<int> &korean){
    int n = russian.size(),wins=0;
    
    // 정렬
    multiset<int> ratings(korean.begin(),korean.end());

    for(int rus = 0;rus < n;rus++){
        // russian[rus]이 한국팀 최고 레이팅보다 큰 경우
        if(*ratings.rbegin() < russian[rus]){
            // 가장 레이팅이 낮은 선수와 경기
            ratings.erase(ratings.begin());
        // 이 외의 경우 이길 수 있는 선수 중 가장 레이팅이 낮은 선수와 경기
        }else{
            ratings.erase(ratings.lower_bound(russian[rus]));
            ++wins;
        }
    }
    
    return wins;
}
