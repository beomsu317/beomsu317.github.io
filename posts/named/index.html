Named

자료형(Type)만으로 의존성을 식별하기 어려운 경우 @Named 어노테이션을 사용해 의존성을 주입한다.

@Named는 Dagger2에서 제공하는 어노테이션이지만 Named는 @Qualifier로 annotate 되어 있다. Named의 정의는 다음과 같다.

1

8

@Qualifier
@Documented
@Retention(RUNTIME)
public @interface Named {

    /** The name. */
    String value() default "";
}

Dagger2에 제공되는 Named 클래스에서 중요한 것은 @Qualifier 어노테이션이다. @Qualifier는 javax.inject 패키지에서 제공되는 어노테이션으로 Named는 @Qualifier로 annotate 되어있는 구조이다.

즉, @Qualifier는 provides 메서드 구분을 위한 커스텀 Named 어노테이션 클래스를 만들기 위한 어노테이션이다.

Example

기본적으로 @Inject 필드 변수에 의존성을 주입하는 방법은 다음과 같다. name 속성을 갖는 Cat이라는 클래스가 있다고 하자.

1

class Cat(val name: String)

다음은 Cat의 의존성을 주입하는 CatModule이다. 같은 Cat 타입을 반환하는 @Provides 메서드가 존재하는데, 서로 name 속성이 다른 객체이다.

class CatModule {
    @Provides
    fun provideGarfield(): Cat = Cat("Garfield")

    @Provides
    fun provideHelloKitty(): Cat = Cat("Hello Kitty")
}

결과는 다음과 같이 CatModule의 @Provides 메서드인 provideGarfield()와 provideHelloKitty()가 같은 Cat 타입이기 때문에 Dagger는 어떤 메서드로 의존성을 주입할지 모르게 되어 오류가 발생한다.

이렇게 반환 타입이 같은 @Provides 메서드가 여러개 존재할 경우 구분을 위해 @Named 어노테이션을 사용한다.

동일한 반환 타입의 @Provides를 사용하기 위해 @Named 어노테이션을 사용하였다.

@Module
class CatModule {

    @Provides
    @Named("Garfield")
    fun provideGarfield(): Cat = Cat("Garfield")

    @Provides
    @Named("HelloKitty")
    fun provideHelloKitty(): Cat = Cat("Hello Kitty")
}

MainActivity에 의존성을 주입하려면 @field:Named("name")을 같이 선언해 Dagger가 동일한 반환 타입의 @Provides 메서드 중 일치하는 메서드를 구분할 수 있다.

class MainAcitivty : AppCompatActivity() {

즉, @Named 어노테이션은 같은 종류의 반환 타입을 가진 @Provides 메서드들 중 선택해서 종속성을 주입하기 위해 사용되는 어노테이션이다.

Custom Named Class

Custom Scope를 정의하는 방법과 비슷하게 annotation class로 선언해준다. 다음은 Custom Name 클래스이다. 해당 클래스는 annotation 클래스로 선언해주어야 한다.

1
2
3
4

@Qualifier
@MustBeDocumented
@Retention(AnnotationRetention.RUNTIME)
annotation class NamedClone(val value: String = "")

의존성 주입을 하는 Module 내부의 provides 메서드이다. 두 메서드 모두 Apple 타입을 반환하며, 메서드 별 @NamedClone()으로 이름을 붙였다.

1
2
3
4
5
6

@Provides
@NamedClone("Red Apple")
fun provideRedApple(): Apple = Apple("red")

@Provides
@NamedClone("Green Apple")
fun provideGreenApple(): Apple = Apple("green")

다음은 의존성을 주입하는 코드이다. 이전에 구현한 @Named에서 Custom Named인 @NamedClone으로 변경된 차이점이 있다.

@Inject
@field:NamedClone("Red Apple")
lateinit var redApple: Apple

@Inject
@field:NamedClone("Green Apple")
lateinit var greenApple: Apple

Enum

위의 @NamedClone 클래스의 속성인 value가 String 타입으로 느슨한 문자열(@Named로 설정된 문자열에 대문자, 소문자가 모두 일치해야 함)에 의존한다. 다음과 같이 @NamedClone의 value로 Red Apple을 소문자로 red apple을 입력할 경우 컴파일에서 오류가 발생한다.

Red Apple != red apple이기 때문이다.

1
2
3

@Provides
@NamedClone("Red Apple")
fun provideRedApple(): Apple = Apple("red")

1
2
3

@Inject
@field:NamedClone("red apple")
lateinit var redApple: Apple

이 방법은 오류가 발생하기 쉽기 때문에 Enum을 사용한다. 다음은 String을 Enum으로 변경한다.

@Qualifier
@MustBeDocumented
@Retention(AnnotationRetention.RUNTIME)
annotation class NamedClone(val color: AppleType)

// Red, Green 두개의 Type만 가진 Enum 클래스
enum class AppleType {
    Red, Green
}

@NamedClone 어노테이션에 Enum 타입을 사용해 타입을 지정해주었다. 만약 AppleType이 아닌 다른 값이 입려되면 구문오류가 즉시 발생하므로 컴파일 전에 수정이 가능하다.

2

4

@Provides
@NamedClone(AppleType.Red)
fun provideRedApple(): Apple = Apple("red")

@Provides
@NamedClone(AppleType.Green)
fun provideGreenApple(): Apple = Apple("green")
