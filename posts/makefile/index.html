Makefile

Beomsu Lee Jan 31 2021-01-31T00:00:00+09:00
11 min

리눅스에선 make라는 파일을 이용해 Makefile을 읽어 주어진 방식대로 명령어를 처리한다. 따라서 많은 수의 파일들을 명령어 1번으로 처리가 가능하다.

Compile

컴파일이란 소스 코드를 컴퓨터가 이해할 수 있는 어셈블러로 변환하는 과정이다. 간단한 예를 보자.

g++의 -c 옵션을 이용해 main.cc의 Object File인 main.o 파일을 생성한다.

1
2
3

$ g++ -c main.cc
$ ls
bar.cc  bar.h  foo.cc  foo.h  main.cc  main.o

이 상태로는 프로그램을 생성할 수 없다. main.o 파일에는 foo() 함수 호출 내용만 있는 것 뿐이지 foo() 함수가 어디있는지, 어떤 방식으로 동작하는지에 대한 내용은 없기 때문이다.

따라서 main.o, foo.o, bar.o 하나로 합치는 과정이 필요한데 이를 Linking이라고 한다.

Linking

링킹은 서로 다른 파일에 흩어져 있던 함수들을 하나로 묶어 링크해주는 작업이다. 위의 main() 함수에서 foo() 함수가 어디에 정의되어 있는지 위치를 찾고 호출할 수 있게 해주는 기능이다.

다음과 같이 컴파일러에 Object 파일을 전달함으로써 링킹이 수행된다.

1

$g++ main.o foo.o bar.o -o main

make

위의 과정을 나열하면 다음과 같다. 이것을 쉘 스크립트로 작성할 수도 있지만 전체 파일들을 컴파일하게 되지만, make를 사용하면 수정된 일부 파일들만 컴파일되도록 할 수 있다.

1
2
3
4

$ g++ -c main.cc
$ g++ -c foo.cc
$ g++ -c bar.cc
$ g++ main.o foo.o bar.o -o main

Makefile

Makefile은 어떤 조건으로 명령어를 실행할지를 담은 파일이며, make를 터미널에서 실행하면 해당 위치의 Makefile을 찾아서 읽게된다. 다음은 위의 내용을 토대로 만든 Makefile이다. make 명령은 Object 파일의 생성 시간과 소스 코드의 생성 시간을 비교하여 명령어 실행을 결정한다.

foo.o : foo.h foo.cc
		g++ -c foo.cc

bar.o : bar.h bar.cc
		g++ -c bar.cc

main.o : main.cc foo.h bar.h
		g++ -c main.cc

main : foo.o bar.o main.o
		g++ foo.o bar.o main.o -o main

main 파일을 만들기 위해 make main 명령을 실행하면 된다.

1

$ make main
g++ -c bar.cc
g++ foo.o bar.o main.o -o main

Variable

Makefile 내 변수를 정의할 수 있다. CC 변수를 정의하고 $()안에 변수 이름을 지정함으로써 사용할 수 있다.

1
2
3
4

CC = g++

foo.o : foo.h foo.cc
		$(CC) -c foo.cc

변수를 정의하는 방법은 2가지가 존재하는데 =과 :=이다.

=을 사용해 변수를 정의하면, 정의에 다른 변수가 포함되어 있으면 해당 변수가 정의될 때 까지 변수의 값이 정해지지 않는다. 따라서 변수들의 정의 순서로부터 자유롭고 싶을 경우 사용한다.

:=을 사용해 변수를 정의하면, 해당 시점의 변수의 값만 확인한다. 따라서 무한 루프를 피할 수 있다.

위의 내용을 토대로 Makefile을 만들고 make 명령을 수행했다. CC에는 컴파일러 이름을, CXXFLAGS는 옵션을 주로 사용한다.

PHONY

Phony는 가짜의, 허구라는 뜻으로 실제 파일을 뜻하는 것이 아니다. 따라서 파일의 존재 유무와 상관 없이 해당 타겟의 명령을 실행하게 된다.

1
2
3

.PHONY: clean
clean:
		rm *.o temp

Pattern Matching

실제 프로젝트에는 많은 파일들을 다루게 되는데 이 파일들에 대해 빌드 방식을 명시하면 Makefile의 크기가 엄청 커지게 된다. 하지만 Pattern Matching을 통해 특정 조건에 맞는 파일들에 대해 recipe를 작성할 수 있다. 패턴은 prerequisite 부분에만 사용할 수 있다.

1
2

%.o: %.cc %.h
		$(CC) $(CXXFLAGS) -c $<
  • %.o는 .o로 끝나는 파일 이름들이 타겟이다.
  • $<의 경우 prerequisie에서 1번째 파일의 이름에 대응되어 있는 변수이다.

Makefile에서 제공하는 자동 변수들은 다음과 같다.

자동 변수설명
$@타겟 이름에 대응
$<의존 파일 목록에 1번째 파일에 대응
$^의존 파일 목록 전체에 대응
$?타겟 보다 최신인 의존 파일들에 대응
$+$^와 비슷하지만, 중복된 파일 이름들 포함

의존 파일에 main.h가 없고 foo.h, bar.h가 있는 main.o는 표현하기 어렵다.

1
2

main.o : main.cc foo.h bar.h
		$(CC) $(CXXFLAGS) -c main.cc

하지만 자동으로 prerequisite을 만드는 컴파일 옵션인 -MD가 존재하며 해당 옵션을 통해 컴파일 할 경우 main.d 파일이 생성된다.

1
2
3

$ g++ -c -MD main.cc
$ cat main.d
main.o: main.cc /usr/include/stdc-predef.h foo.h bar.h

%.o: %.cc %.h 부분을 .d 파일로 대체하는 방법은 다음과 같다. $(OBJS:.o=.d)는 OBJS에서 .o로 끝나는 부분을 .d로 모두 대체하라는 의미이다.

1
2
3
4

CC = g++
CXXFLAGS = -Wall -O2
OBJS = foo.o bar.o main.o

%.o: %.cc
		$(CC) $(CXXFLAGS) -c $<

main : $(OBJS)
		$(CC) $(CXXFLAGS) $(OBJS) -o main

.PHONY: clean
clean:
		rm -f $(OBJS) main

-include $(OBJS:.o=.d)

CC = g++

CXXFLAGS = -Wall -O2

LDFLAGS =

SRC_DIR = ./src

OBJ_DIR = ./obj

TARGET = main

SRCS = $(notdir $(wildcard $(SRC_DIR)/*.cc))

OBJS = $(SRCS:.cc=.o)

OBJECTS = $(patsubst %.o,$(OBJ_DIR)/%.o,$(OBJS))
DEPS = $(OBJECTS:.o=.d)

# 더미 타겟으로 1번째 타겟으로서 작용함
all: main

$(OBJ_DIR)/%.o : $(SRC_DIR)/%.cc
		$(CC) $(CXXFLAGS) -c $< -o $@ -MD $(LDFLAGS)

$(TARGET) : $(OBJECTS)
		$(CC) $(CXXFLAGS) $(OBJECTS) -o $(TARGET) $(LDFLAGS)

.PHONY: clean all
clean:
		rm -f $(OBJECTS) $(DEPS) $(TARGET)

# -include의 경우 .d 파일들을 Makefile에 포함하라는 의미
-include $(DEPS)

patsubst 함수는 $(patsubst 패턴, 치환 후 형태, 변수)와 같이 사용한다. 따라서 위의 경우 $(OBJS)의 모든 %.o 패턴을 $(OBJ_DIR)/%.o로 치환하하는 의미이다.

Multi Core Compile

-j 옵션을 추가하여 make의 속도를 높일 수 있다. 리눅스의 경우 다음 명령을 실행하면 컴퓨터의 현재 코어 개수로 치환되어 실행된다.

1
2
3
4
5

$ make -j$(nproc)
g++ -Wall -O2 -Iinclude/ -c src/foo.cc -o obj/foo.o -MD 
g++ -Wall -O2 -Iinclude/ -c src/main.cc -o obj/main.o -MD 
g++ -Wall -O2 -Iinclude/ -c src/bar.cc -o obj/bar.o -MD 
g++ -Wall -O2 ./obj/foo.o ./obj/main.o ./obj/bar.o -o main
