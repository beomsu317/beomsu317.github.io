# 방법론(Methodology) 심층 리뷰 규칙

방법론 섹션은 리뷰의 핵심이므로 **단순 요약이 아니라 작동 원리와 설계 의도까지 해설**합니다.

## 필수 포함 항목

| 항목 | 설명 |
|------|------|
| **문제 정의** | 이 방법론이 해결하려는 구체적 문제를 한 문단으로 명시 |
| **핵심 아이디어** | 직관적 설명 → 수식 순서로 전개 |
| **구성 요소 분해** | 모델/알고리즘을 세부 컴포넌트로 나눠 각각 설명 |
| **수식 전개** | 수식을 그대로 옮기지 않고, 각 항의 의미와 역할을 문장으로 해설 |
| **설계 선택의 근거** | "왜 이렇게 설계했는가"를 논문 근거를 들어 설명 |
| **한계 및 가정** | 방법론이 전제하는 가정, 적용 범위의 제약을 명시 |

## 서술 순서

1. **동기(Motivation)**: 기존 방법의 어떤 문제를 해결하려 하는가
2. **전체 구조 개요**: 한 문단 또는 그림으로 큰 그림 제시
3. **세부 컴포넌트**: 각 구성 요소를 하위 섹션(`###`)으로 나눠 설명
4. **수식 및 알고리즘**: 핵심 수식을 KaTeX 블록으로, 알고리즘은 의사코드 또는 해당 분야 언어로
5. **핵심 메커니즘**: 분야에 따라 아래 항목 중 해당하는 것을 포함
   - AI/ML: 학습 목표(Loss), 추론 절차
   - 시스템/OS: 동시성 제어, 장애 처리, 성능 모델
   - 네트워크/분산: 프로토콜 상태 전이, 일관성 보장 방식
   - 알고리즘/이론: 복잡도 분석($O(\cdot)$), 정확성 증명 요약
   - 보안: 위협 모델(threat model), 공격/방어 메커니즘
6. **한계 및 가정**: 방법론의 전제 조건과 적용 범위 제약

## 수식 해설 방식

수식은 반드시 아래 패턴으로 설명합니다.

```
수식 제시 → 각 기호 정의 → 전체 의미 한 문장 요약 → 직관적 해석
```

예시 (알고리즘/이론 분야):

> 퀵정렬의 평균 시간 복잡도는 다음과 같이 유도됩니다.
>
> $$T(n) = 2T\!\left(\frac{n}{2}\right) + O(n)$$
>
> 여기서 $T(n)$은 크기 $n$인 입력에 대한 실행 시간, $O(n)$은 피벗을 기준으로 배열을 분할하는 비용입니다.
> 마스터 정리(Master Theorem)를 적용하면 $T(n) = O(n \log n)$이 됩니다.
> 직관적으로는 "각 단계에서 문제를 절반으로 나누고 $\log n$번 반복하므로 전체 비용이 $n \log n$"으로 이해할 수 있습니다.

## 알고리즘 표현

논문에 알고리즘 박스가 있으면 **의사코드 또는 해당 분야 언어**로 재표현합니다. 단, 구현 코드는 논문의 설명을 보조하는 수준으로만 작성합니다.

```c
/* 예: 시스템 논문의 락-프리 큐 enqueue (C) */
bool enqueue(Queue *q, int val) {
    Node *node = malloc(sizeof(Node));
    node->val = val;
    node->next = NULL;
    Node *tail, *next;
    do {
        tail = q->tail;
        next = tail->next;
        if (next == NULL) break;
        CAS(&q->tail, tail, next);
    } while (!CAS(&tail->next, NULL, node));
    CAS(&q->tail, tail, node);
    return true;
}
```

## 자가 점검

방법론 섹션 작성 후 아래를 확인합니다.

- [ ] 수식의 모든 기호가 정의되었는가
- [ ] "왜 이렇게 설계했는가"에 대한 설명이 포함되었는가
- [ ] 단순 나열이 아니라 구성 요소 간 관계가 설명되었는가
- [ ] 그림/표가 있으면 본문에서 참조하고 해설했는가
- [ ] 분야에 맞는 핵심 메커니즘이 설명되었는가 (Loss/추론, 동시성, 프로토콜, 복잡도, 위협 모델 등)
